3.1.2   发布基础影像服务

基础影像服务通常作为应用的底图来使用。一般情况下，覆盖区域较大，数据容量也较大，为了让基础影像服务访问更快，通常会使用多级瓦片缓存技术。

基础影像服务相比影像服务，在iCenter内部能支持更多操作，比如基础影像局部更新，具体参见“基础影像服务管理”章节。

支持发布基础影像服务的数据类型有四种：

GV MBTiles：MBTiles是由Mapbox公司制定的一种基于SQLite数据库的地图瓦片存储开源标准格式，GV MBTiles在MBTiles基础上进行了优化改进。

GV HBase：基于HBase存储组织的一种自定义瓦片存储格式。

TIF目录：由多个TIF文件组成的文件目录。

TIF/IMG文件：.tif或.img格式的单个影像文件。

以上几种类型的数据具体发布方法如下：

在“服务管理”页面点击“发布服务”按钮，进入快速发布服务向导，服务类型选择“基础影像服务”，然后选择对应的数据源类型，通过向导发布即可。

其中，选择具体的数据源类型后，就点击“选择影像数据”按钮可以在服务器上选择具体要发布的数据。

1）发布GV MBTiles

       GV MBTiles数据的组织结构如下图所示：

       {DataDir}

              {0级目录}

                     tiles_x_y.sqlite

              {1级目录}

                     tiles_x_y.sqlite

              {N级目录}

                     tiles_x_y.sqlite

tiles_x_y+1.sqlite

              layer.properties

 

一份GV MBTiles存储了一个图层的瓦片数据，每一组tiles.sqlite存储了对应层级的全部瓦片集合，每一个tiles.sqlite最多存500*500张瓦片。

数据目录下的layer.properties文件存了该图层瓦片的元信息，包括内容如下：

       bounds=-180,-90,180,90

minZoom=0

maxZoom=12

gridset=EPSG:4326

mimetype=png

 

在发布向导中选择影像数据时，选中{DataDir}目录发布即可。

 

2）发布GV HBase

在弹出的GV HBase数据列表中选择要发布的数据即可。

 

3）TIF目录

直接选择存储多个TIF文件的目录即可。注意目录内的TIF文件不要嵌套在多级目录中。

4）TIF/IMG 文件

直接选择服务器上的任意TIF或IMG格式的影像数据文件。

 

 

3.1.3   发布影像服务

发布普通的影像服务需要先将影像数据(TIF或IMG格式)在“数据管理”页面添加到iCenter中，然后通过点击对应数据后面的“发布”按钮进行发布。

首先，通过“从本地上传”或者“从服务器导入”将想发布的影像数据（TIF或者IMG格式）添加到iCenter的“数据管理”页面中，具体步骤参见2.2，2.3节。

然后，在‘’数据管理‘’页面找到添加成功的数据，点击对应的“发布”按钮，进入服务发布向导。点击“下一步”，使用默认参数或者指定想发布的参数，点击“发布”，即可发布成功。

发布过程中可以设置的参数如下：

瓦片格式：PNG或者JPEG。

格网集：EPSG：4326 或者 EPSG：3857。

层级范围：层级的可选范围与格网集有关，具体参见3.1.1节。

透明色：指定某种颜色为透明色，方便处理白边等情况。

立即切片：是否发布服务后立即切片。 默认选择否，可以发布后根据需要手动再切。

发布成功后，可以在”服务管理”对应的影像图层页面，通过指定名称找到该服务，进行服务预览和地址查看。

 

 

3.1.4   发布矢量服务

发布矢量服务需要先将矢量数据(zip包形式的shapefile数据)在“数据管理”页面添加到iCenter中，然后通过点击对应数据后面的“发布”按钮进行发布。

首先，通过“从本地上传”或者“从服务器导入”将想发布的矢量数据（zip包形式的shapefile数据)）添加到iCenter的“数据管理”页面中，具体步骤参见2.2，2.3节。

然后，在‘’数据管理‘’页面找到添加成功的数据，点击对应的“发布”按钮，进入服务发布向导。点击“下一步”，使用默认参数或者指定想发布的参数，点击“发布”，即可发布成功。

发布过程中可以设置的参数如下：

服务名称：指定发布的服务的名称。

字符编码：UTF-8, GBK。

样式表：选择矢量数据使用的样式

瓦片格式：PNG或者JPEG。

格网集：EPSG：4326 或者 EPSG：3857。

层级范围：层级的可选范围与格网集有关，具体参见3.1.1节。

立即切片：是否发布服务后立即切片。 默认选择否，可以发布后根据需要手动再切。

发布成功后，可以在”服务管理”对应的矢量图层页面，通过指定名称找到该服务，进行服务预览和地址查看。

 

3.1.5   发布地形服务

iCenter支持发布的地形数据主要是CESIUM的Terrain格式，具体包括heightmap和quantized-mesh，通过“服务管理”页面的“发布服务”按钮进入发布服务向导进行发布。

发布服务向导目前只支持GV MBTiles形式的Terrain数据。这种格式的数据需要通过数据处理服务由Tif格式的数据来生成。

下面举例说明如何将一个地形文件dem.tif发布为地形服务，共两个步骤。

第一步: 将TIF格式的地形数据处理为GV MBTiles(Terrain)格式

1.      进入服务管理-数据处理，点击新建任务；

2.      数据类型选择” 地形数据”，数据源选择” TIF文件”，瓦片存储选择“GV MBTiles(Terrain)”;

3.      点击选择数据，从服务器上选择地形数据文件” dem.tif”;

4.      点击存放位置，设置数据处理完成后的存放位置；

5.      点击下一步设置参数，输入任务名称，其他参数使用默认值即可，

6.      点击创建任务完成创建。

7.      在数据处理列表中可以看到刚刚创建的任务，等待任务完成后，转到服务发布页面。

第二步：将处理好的GV MBTiles(Terrain)格式的地形数据发布为地形服务

1.      进入服务管理-图层管理-地形，点击发布服务；

2.      服务类型选择“地形服务”，数据源选择“GV MBTiles(Terrain)”;

3.      点击选择数据，从服务器上选择第一步中处理完成的数据。

4.      点击下一步设置参数，使用默认值即可。

5.      点击发布服务，服务发布完成。

 

3.1.6 发布倾斜摄影服务

GEOVIS iCenter支持发布OSGB和Cesium 3DTiles格式的倾斜摄影数据，通过“服务管理”页面的“发布服务”按钮进入发布服务向导来进行发布。

l  如何发布3DTiles格式的倾斜摄影数据？

操作步骤如下：

1.      进入服务管理-图层管理-倾斜摄影页面，点击发布服务。

2.      服务类型选择“倾斜摄影服务”，数据源选择“3DTiles”

3.      点击选择数据，从服务器上选择3DTiles格式的倾斜摄影数据；

4.      点击下一步设置参数，输入图层名称，其他参数使用默认值即可；

5.      点击发布服务，服务发布完成。

l  如何发布OSGB格式的倾斜摄影数据？

OSGB格式的倾斜摄影数据的发布方法和步骤与3D Tiles类似，只需在第2步选择数据源时选择OSGB即可。

3.2.1   基础影像服务管理

进入服务管理页面，在左侧分类中选择图层管理-基础影像，即可查看已经发布的基础影像服务。列表中列出了每个基础影像服务的名称、瓦片格式、格网集、层级范围和切片状态。

l  如何更新基础影像服务

GEOVIS iCenter支持对基础影像服务进行局部更新，操作步骤如下：

1.      找到要更新的基础影像服务，点击更多，在下拉列表中选择更新，进入基础影像服务更新向导。

2.      点击选择数据，在弹出的对话框中选择用来更新基础影像服务的影像数据，选择确定。点击进入下一步。

3.      设置服务参数，使用默认值即可，点击更新服务完成更新。

3.3  服务使用

iCenter通过标准化的服务来共享其管理的各种空间数据。iCenter管理的空间数据及其对应的能发布的服务类型如下表所示：

 

数据类型

支持发布的服务类型

基础影像

TMS、WMS、WMTS

影像

TMS、WMS、WMTS

矢量

TMS、WMS、WMTS、WFS

地形

Cesium Terrain

倾斜摄影

Cesium 3DTiles、OSGB

地名

REST

      

WMTS和WMS是OGC（Open Geospatial Consortium）发布的地图瓦片和地图的服务标准协议。使用支持这两种标准的客户端或者按照协议规范即可使用该服务，应用广泛。

TMS是OSGeo（OpenSource Geospatial Fundation）发布的地图瓦片服务标准协议。使用支持这种标准的客户端或者按照协议规范即可使用该服务。

应用相对来说没有WMTS广泛。推荐使用WMTS服务。

WFS是OGC（Open Geospatial Consortium）发布的要素的服务标准协议。使用支持这种标准的客户端或者按照协议规范即可使用该服务。

通过WFS服务可以获取和查询背后的矢量数据的每个Feature的具体几何和属性信息。

Cesium Terrain和Cesium 3DTiles是按照CesiumJS的数据标准组织的服务，方便Cesium和基于Cesium内核的第三方web客户端直接访问和使用iCenter里管理的地形数据和倾斜摄影数据。

OSGB是倾斜摄影数据的一种常用的数据格式。方便支持OSGB的第三方客户端直接读取iCenter中的倾斜摄影数据。

iCenter内置了一套地名数据，以Rest服务的形式对外提供。iCenter采用Swagger的形式来描述REST服务的API，地名服务的REST API说明可以访问地址：

http://{地名服务host}:8150。通过地名服务的接口可以根据各种空间条件查询到指定地名信息。

 

3.4  样式表的管理和使用

iCenter主要支持通过SLD（Styled Layer Descripter）来指定矢量数据服务化后的样式。SLD的详细规范参见（http://www.opengeospatial.org/standards/sld）。

在“服务管理”的“样式管理”页面可以对iCenter中所有的样式进行管理。

如何添加样式？

在“样式表管理”中点击“添加样式表”按钮可以将一个外部的SLD文件添加到iCenter中。

通常情况下，用户可以选用第三方的桌面工具配置矢量数据的样式风格，比如uDig、QGIS等，然后将配置好的风格导出成SLD文件。

通过该操作将导出的SLD文件添加到iCenter中即可在iCenter中发布矢量数据的时候直接使用。

如何编辑和预览样式？

在“样式表管理”页面找到需要预览和编辑的样式，点击操作区域的“编辑”按钮，在预览图层中可以选择指定图层数据来使用当前样式进行预览。

iCenter提供了样式编辑器可以直接编辑SLD文档的内容，可以支持用户方便的修改SLD中的局部内容，比如填充颜色值等。修改好可以点击“校验”按钮自动检查SLD的内容语法是否正确。

校验成功后点击“保存”，即可看到新样式的效果。

 

3.5  图层组的管理和使用

图层组是把多个图层按一定顺序组合在一起的一个容器，一个图层组可以通过一个服务就将包含的各个图层数据组成的地图统一共享给应用使用。

很多情况下，一幅配好的地图由多个图层组成，有固定的图层叠加顺序。如果没有图层组的能力，客户端需要通过多个服务分别访问多个图层的内容，在客户端再按指定的顺序再进行图层叠加展示。

使用图层组极大的简化了这一过程，减少了对服务器请求的次数，访问带宽压力，同时降低了对客户端（服务消费者）进行图层管理的要求。

 

3.6  数据处理

GEOVIS iCenter 支持将瓦片类型的影像数据和地形数据发布为基础影像服务和地形服务。如果用户只有单个或多个TIF类型的数据文件，需要先通过数据处理服务将TIF数据处理为瓦片数据。

●  如何将TIF影像数据处理为瓦片

1.      进入服务管理-数据处理页面，点击新建任务。

2.      数据类型选择“影像数据”，数据源视情况选择“TIF文件”或“TIF目录”，瓦片存储选择“GV MBTiles”；

3.      点击选择数据，选择要处理的数据或目录；

4.      点击选择存放位置，选择处理完成后瓦片数据的存放位置；

5.      点击下一步设置参数，输入任务名称，其他参数使用默认值即可，

6.      点击创建任务完成创建。

●  如何将TIF地形数据处理为瓦片

步骤与处理影像数据类似，只需在第2步瓦片存储选择“GV MBTiles(Terrain)”,第3步选择对应的地形数据/目录即可。
